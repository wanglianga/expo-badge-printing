# 展会观众证件打印系统

## 原始需求

> 请实现展会证件打印的连续流程：第一步预登记（展示已预登记观众列表，支持选择或现场补录），第二步选择证件类型（普通/ VIP/ 媒体/ 展商），第三步加入打印队列并生成证件。三个步骤需以步骤条或向导形式串联，而非分散字段展示，每一步完成后自动进入下一步。

## 项目简介

ExpoBadge Pro 是一个面向展会场景的观众证件打印前端系统。采用三步连续向导式流程：**预登记（含现场补录） → 证件类型选择 → 打印队列确认**，实现从观众信息检索到证件输出的全流程闭环。

核心业务特性：
- 三步连续流程，支持每步回退修改，完成后自动进入下一步
- 第一步集成预登记列表选择与现场补录两种模式，可灵活切换
- 四种证件类型：普通证、VIP证、媒体证、展商证
- 重复入场权限控制黑名单拦截逻辑
- 演示场景一键触发（预登记样例 / 证件类型失败 / 黑名单历史）
- 所有业务数据持久化到浏览器 localStorage
- 响应式设计，适配桌面与移动端

## 技术栈

- **框架**: Svelte 4
- **构建工具**: Vite 5
- **数据存储**: 浏览器 localStorage
- **样式方案**: 原生 CSS + CSS 变量

## 启动方式

### 前置要求

- Node.js >= 18
- npm 或 pnpm
- Docker / Docker Compose（使用 Docker 启动时需要）

### Docker 一键启动（推荐）

#### 1. 启动服务

```bash
docker compose up --build
```

后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问地址

访问地址：http://localhost:5173

#### 3. 停止服务

```bash
docker compose down
```

### 本地开发启动

#### 1. 安装依赖

```bash
npm install
```

```bash
npm run dev
```

访问地址：http://localhost:5173

```bash
npm run build
npm run preview
```

预览访问地址：http://localhost:4173

## 功能说明

### 三步流程

1. **预登记（含现场补录）**：
   - 支持搜索已预登记观众（姓名 / 编号 / 手机号 / 公司）
   - 点击预登记记录自动切换到表单模式并填充信息
   - 支持直接进入「现场补录信息」模式填写新观众
   - 配置「重复入场权限」（影响黑名单校验）
   - 信息确认后自动进入第二步

2. **证件类型**：
   - 从普通证 / VIP证 / 媒体证 / 展商证 四种证件中选择
   - 根据预登记信息自动推荐证件类型
   - 支持触发库存不足演示场景
   - 选择确认后自动进入第三步

3. **打印队列**：
   - 证件信息预览（含可视化证件卡片）
   - 黑名单校验弹窗（未开通重复入场权限时拦截）
   - 加入打印队列，记录操作历史
   - 打印成功后可开始新的打印流程

### 演示场景触发按钮

顶部导航栏提供三个演示场景快捷入口：

| 按钮 | 说明 |
|------|------|
| ▶ 预登记样例 | 自动加载「张明远」预登记记录并进入第二步 |
| ⚠ 证件类型失败 | 进入第二步后可通过页面内按钮触发「库存不足」失败场景 |
| ⛔ 黑名单历史 | 打开侧栏，查看操作历史、黑名单列表、打印队列 |

### 业务规则

- **重复入场 + 黑名单**：观众在黑名单中但开启了重复入场 → 弹出确认框，可选择继续
- **无重复入场 + 黑名单**：观众在黑名单中且未开启重复入场 → 直接拦截，无法生成证件
- 所有操作（成功打印、黑名单拦截）都会写入操作历史
