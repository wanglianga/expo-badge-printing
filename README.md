# 展会观众证件打印系统

## 原始需求

> 展会观众证件打印前端需要把预登记、现场补录、证件类型、打印队列做成连续流程，而不是分散的字段展示。如果展会观众证件打印前端里的重复入场缺失，黑名单不能继续生成最终结果。启动后先跑预登记样例，再触发证件类型失败，最后用黑名单确认历史记录。 
>  使用Svelte实现即可，要求整体样式符合展会类型的风格，需要的业务数据放到本地存储。

## 项目简介

ExpoBadge Pro 是一个面向展会场景的观众证件打印前端系统。采用四步连续向导式流程：**预登记查询 → 现场补录 → 证件类型选择 → 打印队列确认**，实现从观众信息检索到证件输出的全流程闭环。

核心业务特性：
- 四步连续流程，支持每步回退修改
- 重复入场权限控制黑名单拦截逻辑
- 演示场景一键触发（预登记样例 / 证件类型失败 / 黑名单历史）
- 所有业务数据持久化到浏览器 localStorage
- 响应式设计，适配桌面与移动端

## 技术栈

- **框架**: Svelte 4
- **构建工具**: Vite 5
- **数据存储**: 浏览器 localStorage
- **样式方案**: 原生 CSS + CSS 变量

## 启动方式

### 前置要求

- Node.js >= 18
- npm 或 pnpm
- Docker / Docker Compose（使用 Docker 启动时需要）

### Docker 一键启动（推荐）

#### 1. 启动服务

```bash
docker compose up --build
```

后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问地址

访问地址：http://localhost:5173

#### 3. 停止服务

```bash
docker compose down
```

### 本地开发启动

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发服务

```bash
npm run dev
```

访问地址：http://localhost:5173

#### 3. 生产构建与预览

```bash
npm run build
npm run preview
```

预览访问地址：http://localhost:4173

## 功能说明

### 四步流程

1. **预登记查询**：通过姓名 / 编号 / 手机号 / 公司搜索已预登记观众，或选择直接现场补录
2. **现场补录**：完善观众信息，配置「重复入场权限」（影响黑名单校验）
3. **证件类型**：从 VIP / 专业 / 现场 / 媒体 / 参展商 五种证件中选择，支持触发库存不足演示
4. **打印队列**：证件预览、黑名单校验弹窗、加入打印队列、记录操作历史

### 演示场景触发按钮

顶部导航栏提供三个演示场景快捷入口：

| 按钮 | 说明 |
|------|------|
| ▶ 预登记样例 | 自动加载「张明远」预登记记录并进入第二步 |
| ⚠ 证件类型失败 | 进入第三步后可通过页面内按钮触发「库存不足」失败场景 |
| ⛔ 黑名单历史 | 打开侧栏，查看操作历史、黑名单列表、打印队列 |

### 业务规则

- **重复入场 + 黑名单**：观众在黑名单中但开启了重复入场 → 弹出确认框，可选择继续
- **无重复入场 + 黑名单**：观众在黑名单中且未开启重复入场 → 直接拦截，无法生成证件
- 所有操作（成功打印、黑名单拦截）都会写入操作历史
